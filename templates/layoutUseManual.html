Manual de Regras de Estrutura no Firebase

Este Manual irá mostrar como estruturar o seu banco no Firebase Realtime Database da maneira correta, é importante seguir todos os passos detalhados para evitar erros no uso.
Passo 1 - Criação do Projeto
Criar um projeto no Firebase é bem simples, o que você precisa fazer é:

1.1 - Acessar o site oficial da plataforma. Firebase WebSite
1.2 - Após acessar a página basta logar usando sua conta de Gmail.
1.3 - Já logado a plataforma vá na opção "Ir Para o Console" como mostra a figura abaixo.


Fonte: Próprio Autor.


1.4 - No conlose, clique para criar um novo projeto.


Fonte: Próprio Autor.


1.5 - Basta escolher um nome válido, aceitar os termos de uso e pronto, seu projeto será criado.


Fonte: Próprio Autor.


1.6 - Abra o seu projeto, no Overview, clique em "Database" e criei um banco de dados em tempo real.

Fonte: Próprio Autor.
Passo 2 - Estrutura de Dados
Para a publicação de dados na plataforma, os seus dispositivos devem publicar obedecendo a estrutura básica de dados no banco, importante salientar que a estrutura deve ser seguida a risca, qualquer diferença irá enfluenciar no funcionamento.

2.1 - A identificação dos dispositivos é feita pela palavra chave "ESP-" seguida de um número para identificação, exemplo "ESP-0", essa palavra será utilizada referenciar aquele dispositivo, também será uma "Child" (Raiz) no banco de dados. Todos as "Childs" devem ser filhas da "Child" pai chamada de "Nodes".


Fonte: Próprio Autor.


2.2 - A estrutura deve contemplar algumas informações essenciais, como: Hora e a Data na qual os dados são obtidos pelos dispositivos sensores da aplicação, modo de operação (Opcional) e a região em que está essa sensor (Opcional) e o mais importante, uma lista com os dados de cada sensor e o tipo do dado.
